## ✨ Features

`unprof` transitions raw console logging matrices into a highly clean, robust, and safe structural administration workspace platform.

* 🎨 **Graphical Desktop Interface:** Designed using native Python `Tkinter` window structures with custom layout geometry management matrices for effortless, lightweight window rendering.
* 🏛️ **Advanced OOP Architecture:** Fully demonstrates the intermediate programming tier, decoupling base profiles using hierarchical inheritance patterns linked by standard core mapping protocols.
* 🛡️ **Encapsulation & Safety Shields:** Built-in `try-except` logical validation runtime block arrays that check against faulty grades or text strings to protect systems from crashes.
* 📊 **Dynamic Data Processing:** Automated tracking logic maps subject counts, dynamically scales raw records, and computes precise floating cumulative performance grades in real-time.

## 📚 Documentation Index

To keep the repository clean, readable, and ready for deployment tracking sheets, individual system details have been structuralized below:

<details>
<summary>🏛️ Architecture & Inheritance Mapping</summary>
<br>

The application utilizes a clean separation of concerns separating underlying structural business models from visual components:
* `Member` (Parent Blueprint): Dictates general entity properties including baseline identifiers and master profile string references.
* `Student` (Child Blueprint): Extends parent configurations through the explicit calling of `super().__init__()` parameters and encapsulates personal evaluations within instance dictionary stores.
</details>

<details>
<summary>⚡ Runtime Optimization & Memory Allocation</summary>
<br>

* **In-Memory Store:** Uses dynamic mapping dict arrays linked to runtime hash strings for real-time `O(1)` object lookup profiles, avoiding slower iterative scan routines.
* **Native Component Mapping:** Deployed strictly on top of built-in Python system tools, guaranteeing zero performance lag during system window initialization sequences.
</details>

<details>
<summary>❓ Error Handling & Input Validation</summary>
<br>

* **Numerical Threshold Guards:** Validates grade parameters through tight numerical limits (`0.0` through `100.0`), dropping faulty outlier evaluations before memory commits.
* **Type-Cast Parsing:** Protects parsing routines by filtering raw GUI strings through active error handlers to eliminate unexpected crashes.
</details>

## 🚀 Getting Started & Installation

Get your local instance of the application running on your operating system workspace in 3 simple steps:

### Prerequisites
* Python 3.8 or higher installed. No external packages (`pip install`) are required as the system uses native core packages.

### Local Initialization

# 1. Clone the project repository workspace
git clone [https://github.com/github_username/unprof.git](https://github.com/github_username/unprof.git) && cd unprof

# 2. Confirm your local Git reference settings
git remote -v

# 3. Launch the standalone desktop application runtime environment
python student_system_ui.py



## 💻 Usage Flow

| Step | Action Matrix | Description | Target Container |
| :---: | :--- | :--- | :--- |
| **1** | **Build the Registry** | Input basic alpha-numeric name titles and unique identification codes to instantiate records. | `Section 1: Add Student Details` |
| **2** | **Assign Subject Marks** | Pick an active, live-synced ID from the dropdown selection tree, specify the target course string, and log numeric parameters. | `Section 2: Add / Update Grades` |
| **3** | **Compile Metrics** | Fire execution triggers to compute dynamic floating points, calculate cumulative metrics, and generate text reports. | `Section 3: View Report Card` |
